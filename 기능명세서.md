## 0. 한 줄 정의 (발표용)

> 본 서비스에서 AI는 병역처분을 **예측하지 않고**, 병무청 공공데이터·공개자료를 근거로 **병역판정 준비(건강 백분위)·모집병 특기 탐색·병역진로 상담·민원 분류**를 지원한다. 근거가 부족하거나 개인 판정·예측이 필요한 질문에는 **답변을 거부하는 환각 방지 구조**를 적용한다. 결과물은 4개 탭이 아니라 **입력 1회 → "나의 병역 준비 카드" 한 장**(① 건강 백분위 ② 지원가능 특기 ③ 준비 로드맵 ④ 후속 상담)으로 통합되어 있다.

**차별점:** "기능이 많다"가 아니라 —
1. **거부할 줄 아는 AI** — 근거가 없거나 개인판정/예측이면 거부(e2e 위험질문 **거부율 1.0**, n=8).
2. **준-전수·정직한 백분위** — 전국 96,360명·14지방청 결정론 lookup, 범위 밖은 abstain.
3. **병역 실무 연결의 깊이** — 특기 자격충족(결정론·등급검증) + 진로 로드맵.

> 2025 수상작(인피닉·PreSol)은 전부 B2C 추천이고 **출력 신뢰성 검증이 전무**한 반면, 본 서비스는 거부 F1·근거율·분류 macro-F1을 **독립 골드셋으로 실측·공개**한다.

> 제품 형태(확정): 멀티 화면이 아니라 **단일 카드 + 담당자(B2G) 모드 토글**. 구현: `web/index.html`, `web/app.js`.

> ⚠️ '민원 응답'은 **자유서술 민원 자동응답 생성기가 아니라** F5 분류(5범주+미상)와 집계를 뜻한다. 자유질문 응답은 F1 RAG(절차·기준 질문 한정, 위험질문 거부)가 담당한다.

## F1. 근거 기반 병역 상담 RAG ⭐ (Phase 1 — 실 모델 연결 완료)

- **입력:** 자연어 질문 ("신검 전 준비할 게 뭐야?", "모집병 지원 절차는?")
- **출력:** 근거 문장 + 출처 표기 답변 / 또는 **거부**(사유 + "대신 제공 가능 정보" 안내)
- **데이터:** 상담 KB — **현재 16건 시드**(`data/kb_demo.json`, 신검 절차·모집병·사회복무·여비 등, 출처=병무청 안내 8종). 대규모 공개자료 인덱싱은 **Phase 2 확장 예정**.

### AI 구성 (Mock → 실제 교체 완료)
v1의 Mock retriever/LLM을 실제 모델로 교체했다.
- **검색:** bge-m3 dense 임베딩(`rag/embed.py` — transformers `AutoModel` 직접 로드, CLS 토큰 + L2 정규화, 1024차원) 기반 코사인 유사도 검색(`rag/retriever.py` `BgeRetriever`). 한국어 조사/어미 토큰 미스매치를 임베딩으로 해소.
- **생성:** 로컬 **Qwen3-1.7B**(`rag/llm.py` `LocalLLM`, `MMA_LLM_MODEL` 기본 `Qwen/Qwen3-1.7B`) — "근거에 있는 내용만, 개인 신체등급·합격·면제 예측 금지" 시스템 프롬프트로 환각 1차 방어. 오프라인 HF 캐시 사용.
- **reranker:** bge-reranker-v2-m3 재정렬은 구현돼 있으나 **기본 비활성**(`use_reranker=False`, `MMA_RERANK=1`일 때만). **기본 검색 경로는 dense 임베딩 단독**이다.

### 백엔드 모드 (환경변수 `MMA_RAG`)
| 모드 | 동작 | 비고 |
|---|---|---|
| `mock` (기본) | 토큰 겹침 retriever + mock LLM | 빠름·오프라인 테스트 |
| `bge-extractive` | 실 bge-m3 검색 + 1순위 문서 추출 | 생성 환각 0 |
| `bge-llm` | 실 bge-m3 검색 + Qwen 생성 | **측정 기준 백엔드** |

모델 로드 실패 시 **mock 자동 폴백**(데모는 항상 기동, `service._build_rag`가 LLM/reranker를 eager load해 폴백 보장). 디바이스: `MMA_EMBED_DEVICE`/`MMA_LLM_DEVICE`(CUDA 미가용 시 CPU fp32 폴백).

### 거부 게이트 (4단, `pipeline.py`)
1. **의도 게이트** — 규칙기반 한국어 정규식으로 개인판정/의료진단/합격·면제예측이면 즉시 거부(`gates.intent_gate`).
2. **검색 게이트** — 유사도 임계(`MMA_BGE_THRESHOLD` 기본 0.45) 미달이면 근거부족(NO_EVIDENCE) 거부.
3. **자기일관성 + 부실답변 가드** — LLM 3샘플의 평균 쌍별 **Jaccard(lexical proxy)** 가 임계 미만이면 불확실 거부(bge-extractive 0.6 / bge-llm 0.45 완화).
4. **근거율 게이트** — 답변 토큰의 근거 문맥 등장 비율(lexical proxy)이 임계 미만이면 근거부족 거부.

> 거부 taxonomy(통일): **개인판정 / 의료진단 / 합격·면제예측 / 근거부족** (+ 내부: 불확실). 거부 시 유형별 대체안내(`REFUSAL_MESSAGES`) 제공.
> ⚠️ 자기일관성·근거율은 현재 **lexical proxy(토큰 겹침)** 다. **NLI 기반 faithfulness 고도화는 Phase 2 예정**(코드 주석에도 명시).

### 측정 (end-to-end, `scripts/run_rag_eval.py` → `eval/rag_e2e_report.json`)
검색 → 생성 → 근거율·자기일관성 게이트까지 전체 파이프라인을 22문항으로 측정(backend=bge-llm):

| 항목 | 실측값 |
|---|---|
| 위험·무관 질문 거부(n=8) | **거부율 1.0** (개인판정·의료·예측·무관 전부 차단) |
| 절차 질문 답변(n=14) | **답변율 0.857**, 평균 **근거율 0.897** |
| 전체 정확도(expected 일치) | **0.909** (소요 23.6s ≈ 1.07s/문항) |

> 미답변 2건("자격증이 모집병 지원에 도움이 되나요?", "재검사는 어떤 경우에 신청하나요?")은 자기일관성 게이트가 보수적으로 거부한 결과 — 안전 우선의 정직성 비용이며, **위험질문 거부율 1.0**이 핵심 지표다.

## F2. 모집병 특기 추천 ⭐

전공·자격증·관심분야를 입력하면 지원 가능한 모집병 특기를 추천한다. 핵심은 **합격 예측이 아니라 '자격 검토 가능' 선까지의 정직한 매칭**이며, 검증 가능한 결정론 부분과 휴리스틱 랭킹 부분을 한 점수로 뭉뚱그리지 않는다.

- **입력:** 전공, 자격증, 관심분야, 희망 군(다중)
- **출력:** 추천 특기 + 추천 사유 + **자격 상태 4분류** + 면책 고지
  - `ok` — 보유 자격/전공이 공개 요건과 일치하고 등급·점수까지 검증됨(`qualifies=True`)
  - `unknown` — 명칭은 일치하나 등급·요건을 확정할 수 없음 → **본인 확인 필요**(`verification_required=True`, 거짓 충족으로 승격하지 않음)
  - `grade` — 명칭 일치하나 등급·점수 요건 **미충족 확인**
  - `no_match` — 자격은 미해당, 전공·관심 의미 관련도로만 후보
  - 비보유 가드: "준비/목표/예정/공부" 같은 **희망 표현은 보유로 처리하지 않음**
- **데이터:** **3066750**(전공명·자격면허·등급→군사특기·군명). 전량(약 1.36M행)을 `scripts/fetch_teukgi_index.py`로 **(특기명|군) 단위 사전 집계** → `data/teukgi_index.json` **494 엔트리**(변형 dedup 시 base 특기명+군 372개). 서비스는 이 인덱스를 lazy 로드해 매칭한다(매 질의 전수 스캔 회피).
- **AI 구성:**
  - **(a) 자격 충족 필터 — 결정론·검증 가능.** 자격/전공 **전체 명칭 포함** 매칭(부분 토큰 거짓 충족 차단) + **등급·점수 검증**(예: '토익 900점이상' 같은 점수 내장 자격명 파싱, 기사·산업기사 등 등급 랭크 비교). 코드: `src/mma_navi/recommend/teukgi.py`, `index.py`.

| 항목 | 상태 |

## F3. 건강 리스크 해석 (BMI/신장/체중 백분위)

- **입력:** 신장·체중 (→ BMI는 `compute_bmi`로 직접 계산, `mma_api.py`)
- **출력:** 동년 병역판정 대상자 대비 **BMI·신장·체중 백분위** + 결정론 해설 + 범위 밖 정직 거부
  - 가치중립 표현 사용: "상위 N%"가 아니라 **"백분위 N (나보다 낮은 또래 N%, 높은 또래 N%)"** (BMI 오해 방지, `percentile.py` `as_message`)
  - 모든 출력에 **"백분위 정보이며 신체등급/병역처분 예측이 아닙니다"** 고지 부착
- **데이터(실데이터):** **data.go.kr 3064321**(신검 microdata) **단일 소스**에서 전국 + **14지방청** BMI·신장·체중 분포를 빌드(`scripts/fetch_and_build.py`, `jbceong` 필드로 지방청 분할).
  - 규모: **전국 96,360명**(준-전수), BMI 구간 **18.5~35.0**(공개 API 절단), 코호트 = 전국 + 14지방청(총 15)
  - 산출물: `data/distributions_real.csv` (`metric,cohort,bin_low,bin_high,count`)
- **AI / 엔진:**
  - 숫자 = **파이썬 결정론 분위수 lookup**(`percentile.py`). 학습 모델이 아니라 히스토그램 lookup이므로 **환각 구조적 불가**(같은 입력=같은 출력).
  - 해설 = **결정론 한국어 메시지**(`as_message`, **LLM 아님**). `percentile.py`/`dataio.py` 경로에는 LLM·임베딩 호출이 전혀 없다.
- **정직성 가드레일 (구현·검증 완료):**
  - 데이터 범위 밖/미커버는 숫자를 지어내지 않고 **ABSTAIN**: `BELOW_MIN`(<18.5) · `ABOVE_MAX`(>35) · `NO_COVERAGE`(bin gap) · `NO_DATA`(빈 분포) · `INVALID_INPUT`(NaN/inf)
  - **개인 신체등급/합격/면제 예측 금지**(집계 데이터로 개인 판정은 생태학적 오류) — 엔진은 백분위만 계산
  - 검증: 엔진 테스트 **15/15 통과**(`tests/test_percentile.py`: 절단 거부·gap 거부·NaN 거부·고지문 포함)
- **API:** `POST /api/percentile` → `service.percentile_report` → BMI·신장·체중 3블록 + disclaimer 반환. 미존재 코호트는 전국으로 안전 폴백.

**미구현 / Phase 2:**
- **15117367**(별도 신장 14지방청 연간 CSV) 실연동 — **현재 미사용**. `dataio.py`의 `fetch_distribution_api`는 `NotImplementedError`(파서 TODO)이며, 신장 분포도 **3064321에서 빌드**한다. 별도 데이터셋 통합은 예정.
- 해설 LLM 연동: 현재 F3 해설은 결정론 메시지로 충분히 동작하며, 거부가능 LLM 해설을 별도로 붙이는 것은 예정(상담 RAG와 분리).
- 연도별 추세: 데이터는 2026 단년. 다년 추세는 Phase 2.

## F4. 병역 준비 경로 생성 (접착제 — 규칙 오케스트레이션)

- **입력:** 단일 카드 입력 1회(신장·체중·지방청·전공·자격증·관심사) → F3 백분위 결과 + F2 특기 결과
- **출력:** 5단계 개인 로드맵. 각 단계 = `n / title / text(내 결과 인용) / tone(ok·warn·info) / link(공식 URL) / ask(후속 상담 프리필)`
  1. 병역판정검사 일정·통지 확인
  2. 신체검사 준비 — **내 BMI 백분위 인용**(범위 밖이면 '판단 보류'로 정직 표기, 신체등급 예측 아님 고지)
  3. 모집병 특기 확인 — 자격충족(`ok`)만 '1순위', 등급·요건 미확인(`unknown`)은 '본인 확인 필요'로 **분리**(과대표기 금지)
  4. 현역·보충역 두 경로 모두 준비 — 본 서비스는 결과를 단정하지 않음
  5. 병역진로설계센터 상담(실 CTA)
- **데이터:** F2·F3 산출 결과를 직조(별도 신규 데이터 호출 없음). 링크는 병무청/사회복무 포털 등 **안정 공식 도메인만**(`MMA_LINKS`: `mma.go.kr`·`sbg.mma.go.kr`, 죽은 버튼 방지로 회차별 딥링크 지양).
- **AI:** **규칙 기반 결정론 오케스트레이션**(LLM·학습 없음, 코드 주석 명시). 이미 계산된 percentile/teukgi 결과 dict를 `.get()` 방어 인용해 문장에 엮는 '접착제'. → AI30 무게중심 아님(보조).
- **구현:** `service.py:build_roadmap` / `roadmap_report`(단일 입력→백분위+특기+로드맵 1회 왕복) / `POST /api/roadmap`.
- **실연동:** 로드맵 4단계에 사회복무 복무기관(3066757)·소집계획(3066753)·공석(3066754) 지방청 실데이터, 5단계에 진로설계센터(15148370) 관할 센터, 특기추천에 접수현황(15031295) 경쟁률을 인용(전부 결정론 템플릿, LLM 아님). 'LLM 요약' 단계는 없음.

> **정직성 메모:** 초기 명세의 'LLM 요약'은 미사용. F4는 100% 결정론 규칙이며 '핵심 AI 모델'로 오버셀하지 않는다(서사·UX 접착제). 검사 수치는 절차 안내까지만, 최종 병역처분은 병역판정검사로 결정됨을 로드맵 note에 고정 표기.

## F5. 민원 질문 분류 + 자동분류 집계

- **입력:** 자연어 질문 / **출력:** 카테고리(**신검·모집병·사회복무·여비·진로 5범주**) + 키워드 무적중 시 **'미상' 폴백**
- **AI:** **키워드 argmax(결정론·설명가능)**. `CATEGORY_KEYWORDS` 사전 기반 점수 최대값, 동점 시 카테고리 순서 우선. (임베딩/LLM few-shot 분류는 **미구현 — Phase 2 예정**.)
- **구현:** `classify.py:classify` / `service.py:classify_query` / `POST /api/classify`.
- **민원 유형 자동분류 집계(B2G 인사이트):** `service.py:complaint_stats` / `GET /api/complaints-stats`. 실 민원 로그 부재로 **독립 골드셋 148건**(분류 50+20 · 거부 66+12)을 데모 코퍼스로 사용. 카테고리 분포 + `intent_gate` 자동거부율 산출.
  - 데모셋 148건 기준: **자동거부 26.4%(39건)**, 응답가능 109건.
  - **'데모셋 기준 추정 — 실 민원 로그 아님' 라벨을 응답 note에 고정**(8-B #6 가드레일 준수).
- **측정(AI30):** 분류 정확도·macro-F1을 독립 골드셋으로 측정.

| | dev(튜닝 사용, n=50) | held-out(미사용, n=20) |
|---|---|---|
| accuracy | 0.98 | 0.80 |
| macro-F1 | **0.98** | **0.807** |

> held-out에서 일반화 갭(0.98→0.81)을 정직하게 노출. **'진로' 범주 held-out recall 0.25**(취업가산점·학점인정·기술교육 등 표현 다양성)가 주 오류원 — 키워드 한계로 Phase 2 임베딩 분류 동기.

## F6. (B2G) 14지방청 청년건강 히트맵 ½

- **출력:** 14지방청별 신검 통계 히트맵(중앙값·평균·n) + BMI 시 **과체중+(BMI≥25)·비만(BMI≥30) 비율** + 전국 기준선 + **KNHANES 참고치 오버레이**
- **데이터:** 병무청 병역판정검사(3064321) **준-전수** 분포. 코호트 15개 = 전국 1 + **지방청 14**. 전국 n=**96,360**. (BMI 공개데이터 18.5~35 절단 → 비율은 '범위 내' 기준.)
- **참고 벤치마크:** 질병관리청 국민건강영양조사(KNHANES) 19–29세 남성 **BMI≥25 약 40%**(코드 내 정적 상수 `KNHANES_REF`, `rate_approx=40.0`). 출처·주의(표본조사 vs 준-전수, 측정·표본 기준 차이로 직접 비교 불가, 참고용)를 함께 노출.
- **구현:** `service.py:b2g_heatmap` / `GET /api/b2g?metric=bmi|height|weight`. KNHANES 오버레이·over25/over30 비율은 **`metric=bmi`에서만 활성**(신장·체중은 중앙값·평균·n).
- **용도:** 발표 서사용 1화면(병무청 정책방향 '청년건강 국가통계화'와 연결). AI30 무게중심 아님.
- **예정(Phase 2):** 질병청 15139178 **실 API 연동은 미구현**(KNHANES는 코드 내 참고 상수). **연도별 추세**는 2026 단년 데이터라 미구현(연차 축적 시 추세 분석 가능 — note에 미래형 명시).

> **정직성 메모:** F6은 '질병청 데이터 실시간 통합'이 아니라 '준-전수 신검 데이터 + KNHANES 참고치 병치(표본 vs 준-전수)' 수준임을 명확히 한다. 닿는 척 하지 않는다.

- "답변가능" 정의: 공개 절차·기준 · 공개통계 기반 백분위 · 특기 자격요건.

### 3.2 AI30 측정 결과 (실측, `python scripts/run_eval.py` → `eval/report.json`)

> 거부게이트·분류기를 dev/held-out 골드셋으로 측정. **이 거부 수치는 `intent_gate()` 단독 성능**(검색 근거·자기일관성 미반영)이며, 전체 파이프라인 거부는 3.3의 e2e 거부율 1.0이다 — 두 측정 범위를 구분해 표기한다.

| | 거부게이트 (intent_gate 단독) | 민원분류 |
|---|---|---|
| **dev** (튜닝 사용) | P=1.0 R=1.0 F1=1.0 / 사유 macro-F1 **0.981** (n=66) | acc 0.98 / macro-F1 **0.98** (n=50) |
| **held-out** (미사용) | 과잉거부 1/12, answer 정확도 **0.917** | macro-F1 **0.807** / acc 0.80 (n=20) |

> held-out 일반화 갭을 정직하게 노출한다 — 분류 macro-F1이 dev 0.98 → held-out **0.807**(특히 '진로' 범주 F1 0.40)로 떨어진다. 과장 없이 '측정·검증한 AI'임을 보여주는 증거.

### 3.3 End-to-end RAG (Phase 1, 실 bge-m3 + 로컬 Qwen3-1.7B)

`python scripts/run_rag_eval.py` → `eval/rag_e2e_report.json`. **전체 파이프라인**(검색 → 생성 → 근거율·자기일관성 게이트)을 22문항으로 측정. 거부는 intent_gate가 아니라 4층(의도 + 검색근거없음 + 자기일관성 + 근거율)을 모두 통과해야 답변한다.

| 대상 | 결과 |
|---|---|
| 위험·무관 질문 거부 (n=8) | **거부율 1.0** (개인판정·의료·예측·무관 전부 차단) |
| 절차 질문 답변 (n=14) | 답변율 **0.857**, 평균 근거율 **0.897** |
| 전체 정확도 (expected 일치) | **0.909** (23.6s / 22문항 ≈ 1.07s/문항) |

> 미답변 2건('자격증이 모집병 지원에 도움?', '재검사 신청 경우?')은 자기일관성 게이트가 보수적으로 거부(LLM 3샘플 표현 분기) — 안전 우선의 정직성 비용. 근거율은 별도 faithfulness 라벨셋이 아니라 `grounding_ratio`(토큰 겹침 proxy)로 답변건에 한해 산출한 값임을 밝힌다.

## 4. 가드레일 — AI가 하면 안 되는 것 (코드 레벨 차단)

거부 taxonomy는 `src/mma_navi/rag/gates.py`의 `RefusalReason` enum으로 단일화: **개인판정 / 의료진단 / 합격예측 / 근거부족** (+ 내부 게이트 '불확실').

| 금지 | 이유 | 실제 차단 위치 (코드) |
|---|---|---|
| "당신은 4급입니다" (개인 신체등급) | 개인 판정 불가(집계데이터) | `intent_gate` `_INDIVIDUAL` 패턴 → 개인판정 거부 |
| "면제/합격 가능성 높습니다" | 병역처분=공식검사 | `intent_gate` `_PASS` 패턴 → 합격예측 거부 |
| "질병명·진단 예측" | 의료진단 위험 | `intent_gate` `_MEDICAL` 패턴 → 의료진단 거부 |
| "이 특기 무조건 합격" | 경쟁률·자격 변동 | F2 출력 템플릿 문구 제약(**별도 출력 후처리 룰 모듈은 없음**) |
| 근거 없는 답변·환각 | 출처 없는 생성 | `pipeline.py`: 검색결과 없음→NO_EVIDENCE, 근거율 미달→NO_EVIDENCE |
| 표현이 흔들리는 답변 | 신뢰 불가 | `pipeline.py`: `self_consistency`(3샘플 Jaccard) 미달→불확실 거부 |

- **의도 게이트는 규칙 기반 한국어 정규식**(`intent_gate`, `_MEDICAL`/`_INDIVIDUAL`/`_PASS`). 한국어 `\b` 미신뢰 문제 때문에 '위험 대상어 + 판단/예측 동사' 조합으로 잡고, 절차/안내 질문은 동사를 넣지 않아 false positive를 피한다. 정밀도는 §3 거부 평가셋으로 측정·튜닝.
- **근거/일관성 게이트는 lexical proxy(토큰 겹침)** — NLI 기반 고도화는 **Phase 2 예정**.
- **표준 응답문:** "공개 데이터 기준 참고 정보입니다." / "최종 병역처분은 병역판정검사로 결정됩니다." / "공식 판단은 병무청·병역진로설계센터 상담이 필요합니다."

## 7. 아키텍처 / 기술스택 (실제 구현 기준)

```
[입력 1회]  web/index.html (단일 '나의 병역 준비 카드', vanilla JS)
   │  사용자 모드 ↔ 담당자(B2G) 모드 토글
   ▼
FastAPI  app/server.py  →  app/service.py (순수 함수 레이어)
   ├─ /api/roadmap   : 입력1회 → 백분위+특기+로드맵 (왕복 1회)
   ├─ /api/percentile: 결정론 백분위 엔진 (percentile.py, gap-aware abstain)
   ├─ /api/teukgi    : 자격충족(결정론) + bge-m3 의미 관련도 (recommend/)
   ├─ /api/consult   : 상담 RAG (rag/ — 답하거나 거부)
   ├─ /api/classify  : 민원 5범주 분류 (classify.py, 키워드 규칙)
   ├─ /api/b2g       : 14지방청 청년건강 히트맵 + KNHANES 참고치
   └─ /api/complaints-stats, /api/metrics, /api/status
```

### 7-1. 스택
- **Frontend:** 단일 페이지 "나의 병역 준비 카드"(vanilla JS, 빌드 도구 없음). 4탭이 아니라 **입력 1회 → 한 장**(①건강 백분위 ②특기 ③로드맵 ④후속상담) + **사용자/담당자 모드** 토글. AI신뢰도는 푸터 접이식/담당자 모드로 강등.
- **Backend:** FastAPI(`app/server.py`) + 서비스 레이어 분리(`app/service.py`), lifespan 백그라운드 warmup.
- **데이터층:** data.go.kr 사전 수집 → 로컬 캐시 + **분위수/히스토그램 사전계산**(CSV·JSON 산출물). 런타임 실시간 호출 아님.
- **백분위 엔진:** 순수 파이썬 결정론 lookup(`percentile.py`). 범위 밖·gap은 수치 생성 없이 abstain → **환각 구조적 불가**.

### 7-2. AI 구성요소 (실모델·오프라인)
- **임베딩:** `BAAI/bge-m3`(1024dim, CLS 토큰 + L2정규화). **transformers 직접 로드**(`AutoModel`), FlagEmbedding 미사용. `rag/embed.py`.
- **검색:** `BgeRetriever`(코사인 dense 검색, `rag/retriever.py`). **reranker `BAAI/bge-reranker-v2-m3`는 옵션**(기본 OFF, `MMA_RERANK=1`일 때만).
- **생성:** 로컬 `Qwen/Qwen3-1.7B`(`rag/llm.py`, `AutoModelForCausalLM`) — 시스템 프롬프트로 근거 외 생성·개인 등급/합격/면제 예측 금지.
- **거부 게이트:** `rag/gates.py`+`pipeline.py` — intent(개인판정/의료/합격예측) · 근거율 · 자기일관성 3중 방어.

## 부록 B. 남은 정직성 주의(발표 시 과대인용 금지)

- §5 입력 패널에서 '나이' 필드는 v1 명세에 있었으나 실제 폼에는 없음(키·체중 기반). v2 본문에서 '나이 입력 없음'으로 정정해 표기했으나, 일부 발표 자료/구두 설명이 여전히 '연령대 또래 비교'를 함의하면 미세한 과대해석 여지가 남는다(코호트는 2026 병역판정 대상 연령대 전체로, 개별 연령 입력 기반이 아님).
- e2e RAG 측정 n=22(답변 14·거부 8)는 통계적 표본이 작다. 위험질문 거부율 1.0·답변율 0.857 등은 소표본 점추정이며 신뢰구간이 넓다 — '거부율 1.0'을 '항상 거부'로 일반화하면 과대주장이 될 수 있어 표본수(n)를 항상 병기했다.
- 거부게이트 dev P/R/F1=1.0은 튜닝에 사용한 dev셋 수치로, 같은 데이터에 패턴을 맞춘 결과(과적합 가능)다. v2에서 'intent_gate 단독·dev'임을 명시하고 held-out(과잉거부 1/12)을 병기했으나, 발표에서 '거부 F1=1.0'만 떼어 인용하면 과대주장이 된다.
- 근거율(평균 0.897)은 NLI 기반 faithfulness가 아니라 토큰 겹침 lexical proxy다. '근거율 0.897'을 의미적 사실충실도로 읽으면 과대해석 — v2에서 proxy임을 반복 명시했으나 수치만 인용될 위험이 남는다.
- F2 의미 관련도는 정성 예시('보안'→정보보호병)만 검증됐고 P@k/nDCG 정량 미측정이다. '의미 관련도 랭킹 구현'이라는 표현이 정량 성능을 함의하지 않도록 'Phase 2 정량 측정 예정'을 달았으나, 데모에서 '추천이 정확하다'는 인상은 정량 근거가 없다.
- 상담 KB가 16건 시드에 불과해 커버리지 밖 질문은 근거부족으로 거부된다. 'RAG 상담 기능'이 광범위한 질문에 답한다는 인상은 현 KB 규모로는 과대 — v2에서 KB 16건·Phase 2 확장으로 명시했다.
